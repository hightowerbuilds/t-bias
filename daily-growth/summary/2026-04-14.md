## Session 3 — Three-Phase Roadmap Completion

### Phase 1 — Architecture Restructure
- Introduced `IRenderer` interface (`IRenderer.ts`) to decouple rendering from terminal logic
- Created `ipc/types.ts` — explicit TypeScript ↔ Rust IPC contract with typed command/event names
- Split monolithic terminal into `TerminalCore` (pure logic, no DOM) and `TerminalHost` (DOM orchestrator)
- Layered canvas architecture: Layer 0 (text + backgrounds), Layer 1 (selection overlay), Layer 2 (cursor)
- `RenderState` interface with both `getCell()` compatibility path and zero-allocation typed-array access

### Phase 2 — Canvas 2D Rendering Engine
- Built `GlyphAtlas.ts` — dynamic texture atlas on OffscreenCanvas (1024x1024 pages, up to 4, LRU eviction)
  - Rasterizes each unique glyph once (keyed by char + bold/italic/faint + fg color + wide), blits via `drawImage()`
  - Hit/miss metrics tracking for performance monitoring
- Three-pass row rendering in `CanvasRenderer`: background run-merging → glyph blitting → decorations
- Dirty-row bitmap tracking (`Uint8Array`) — only redraws changed rows, with partial-redraw optimization
- DPR change detection (window moved between displays) with automatic re-measurement and atlas rebuild
- Pixel-snapped cell dimensions to avoid subpixel blending artifacts at fractional DPRs
- Font zoom support (Cmd+/-/0) with atlas cache invalidation

### Phase 3 — Terminal Emulation Core Hardening
- `VirtualCanvas.ts` — Structure-of-Arrays cell buffer (`Uint32Array` for chars/fg/bg/attrs/ulColor)
  - Dual-page buffers (main + alt screen), scrollback with configurable limit
  - Grapheme cluster support via sentinel value (`0xFFFFFFFF`) + `Map<number, string>` side-store
  - Solid.js signals per row for reactivity, batch begin/end for coalesced updates
  - Bulk `copyWithin()` for scroll operations instead of row-by-row copying
- Parser hardening: DCS (Device Control String), APC (Application Program Command), PM (Privacy Message) states
  - Colon sub-parameter support (`SUB_PARAM_MARKER`) for sequences like `SGR 4:3` (curly underline)
- Screen.ts expanded to ~930 lines:
  - Full mouse tracking: modes 1000/1002/1003 with SGR extended encoding (1006)
  - Focus events (mode 1004)
  - DEC graphics charset mapping (G0-G3, SO/SI switching)
  - Tab stops (HTS, TBC, CHT, CBT)
  - Insert mode (IRM), REP (repeat character)
  - Window ops (title push/pop), DECALN, origin mode
  - Underline styles: single, double, curly, dotted, dashed + underline color (SGR 58/59)
  - Overline (SGR 53/55)
- Selection and copy support (`Selection.ts` + Cmd+C handling in TerminalHost)
- Scrollback viewport navigation (Shift+PageUp/Down, mouse wheel)
- Write coalescing: buffer rapid PTY output, flush once per `requestAnimationFrame`
- Debug overlay (Cmd+Shift+D): draw time, dirty rows, atlas stats, input latency, throughput
- Screen state dump (Cmd+Shift+S) for debugging typed-array content
- 28 unit tests (`core.test.ts`) via Vitest covering cursor movement, text output, erase ops, SGR, scroll regions, tab stops, charsets, alt screen, modes, device responses, DECSTR, insert/delete, and typed-array consistency
