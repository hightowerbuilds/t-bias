## Key Decisions

**Why `onpaint` instead of pure RAF for the scroll bug:**
The duplicate-line bug is a race between `scrollUp` (which rotates the row-index map
and marks rows dirty) and the RAF render loop reading that dirty state. With `onpaint`,
the browser guarantees layout is settled before we read `getRenderState()` — no
interleaving possible within a single rendering cycle.

**Why keep `detectHoveredUrl` even with `<a>` elements:**
`cursorCanvas` sits on top and intercepts all mouse events. `<a>` elements inside
`textCanvas` don't receive native mouse events from above. The `<a>` pool is
accessibility-only; the visual pointer cursor and Cmd+Click still come from our
existing hover-detection code.

**Why not remove OSC 8 underline drawing from `drawSelectionLayer`:**
Same reason — the `<a>` elements are transparent and positioned behind `cursorCanvas`.
The underlines must remain in our canvas drawing code to be visible.
