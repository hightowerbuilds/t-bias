### 1. Codebase Review & Dev Server Launch
Explored the full project structure, read through App.tsx, Panes.tsx, pane-tree.ts, Terminal.tsx, and the Rust backend. Launched the Tauri dev server to get the desktop window running.

### 2. FlipExplorer Bug Fix
The FlipExplorer component was crashing on mount with `has@[native code]` error. Root cause: `createStore<Set<string>>(new Set())` — SolidJS stores proxy objects, but `Set` methods require internal `[[SetData]]` slots that proxies can't provide. Fixed by switching to `createSignal<Set<string>>` and updating all accessors.

### 3. Hamburger Flip Button
Added a `☰` button to the top-left corner of terminal panes for flipping to the file explorer (and back). The button appears on both the front (terminal) and back (explorer) faces in the same position. Removed the old "↻ Flip" button from FlipExplorer's header. Button is subtle (35% opacity) and brightens on hover.

### 4. Terminal Text Padding
Changed the default padding from `0` to `8` pixels in `src-tauri/src/config.rs`, giving the terminal text breathing room from the edges.

### 5. CWD Sync — OSC 7 Wiring (Initial Attempt)
Wired OSC 7 (current working directory) end-to-end:
- `Screen.onCwd` → `TerminalCore.handleCwd` (parses `file://` URI) → `TerminalHost.onCwdChange` → `Terminal.tsx` prop → `App.tsx` stores in `paneCwds[paneId]` → `FlipExplorerView` receives as `cwd` prop

This didn't work because macOS zsh only emits OSC 7 when `TERM_PROGRAM=Apple_Terminal`.

### 6. CWD Sync — Pull-Based `get_pane_cwd` (Working Solution)
Planned and implemented a pull-based approach that queries the shell's CWD directly from macOS:

**Rust (`src-tauri/src/pty.rs`):**
- Added `child_pid: Option<u32>` to `PaneState`, captured via `child.process_id()` before the child moves to the exit-wait thread
- Added `get_pane_cwd(pane_id)` Tauri command
- `get_pid_cwd()` uses `libc::proc_pidinfo` with `PROC_PIDVNODEPATHINFO` to read the shell process's CWD — works with any shell, no configuration needed
- Added Linux fallback via `/proc/{pid}/cwd` symlink
- Set `TERM_PROGRAM=t-bias` on PTY spawn (enables OSC 7 as a bonus for future shell integration)

**Frontend (`src/FlipExplorer.tsx`):**
- Added `createEffect` that fires when `props.isActive` becomes true (on flip)
- Invokes `get_pane_cwd` and loads the returned directory into the file tree
- OSC 7 chain kept as a secondary update path

**Result:** `cd` to any directory in the terminal, press Cmd+/ to flip — the file explorer now shows the terminal's current directory.

## Key Decisions

**Why pull-based CWD instead of fixing OSC 7:** macOS zsh's `/etc/zshrc` only emits OSC 7 for `TERM_PROGRAM=Apple_Terminal`. Setting that would be hacky. Using `proc_pidinfo` works universally with any shell and requires zero user configuration.

**Why hamburger on both sides:** Consistent affordance — the flip button is always in the same spot regardless of which face you're looking at.
