# Session Summary — 2026-04-19

## What We Did

### 1. Terminal Trust Hardening — Process Lifecycle (Phase 1)
Addressed the most critical gap in the terminal's reliability: PTY processes surviving app quit.

- Added `PtyState::close_all()` in Rust to terminate every tracked PTY child process
- Restructured `lib.rs` to hook into Tauri's `RunEvent::Exit`, guaranteeing cleanup regardless of how the app exits (Cmd+Q, window close, forced quit)
- Replaced the fire-and-forget `beforeunload` handler with Tauri's `onCloseRequested` so the shell registry shutdown is properly awaited before the window closes
- Defined and documented the lifecycle contract:
  - Pane close: SIGHUP → SIGCONT → SIGTERM → SIGKILL the process group
  - Tab close: close all panes in tab
  - App quit: shell registry updated, then all PTYs terminated by Rust exit handler
  - Background jobs: terminated with the shell (not detached)
  - Shell exits on its own: "[Process exited]" shown, pane stays open

### 2. Close Confirmation Dialog
Added an in-app confirmation modal when closing tabs, panes, or the app while foreground processes are running. Shows the process names (e.g. "vim is still running. Closing will terminate it.") with Cancel/Close options. Styled to match the dark theme.
