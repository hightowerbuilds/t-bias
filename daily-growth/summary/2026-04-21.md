### 7. Terminal Rendering — Synchronized Output (DEC 2026)
Implemented the synchronized output protocol that TUI apps (Claude Code, Gemini, Ink-based tools) rely on to prevent partial frame tearing. When mode 2026 is enabled, the renderer defers canvas draws until the app signals the update is complete. Includes a 200ms safety timeout to prevent frozen display if an app crashes mid-sync.

### 8. Terminal Rendering — Viewport-Aware Scroll
Fixed scrollback rendering — the renderer was always reading from the active screen's typed arrays regardless of viewport position. Added `RowSource` abstraction so each visible row resolves to the correct data source (scrollback ring buffer or active screen). Also fixed tab visibility leak (`visibility: "visible"` → `"inherit"` in Panes.tsx).

### 9. Rust VT Backend (Major Architecture Change)
Migrated VT escape sequence processing from JavaScript to Rust using the `vte` crate (the same parser powering Alacritty). This is the biggest architectural change in the project's history.

**New Rust module (`screen.rs`, ~900 lines):**
- `ScreenBuffer` — complete terminal state machine: cell grid, cursor, scroll regions, SGR (16+256+truecolor), alt screen, scrollback, insert/delete, erase, synchronized output
- Implements `vte::Perform` trait wired to the battle-tested `vte` parser
- `ScreenState` wraps buffer + parser; `process(bytes)` feeds raw PTY output
- `take_frame()` returns dirty-checked `ScreenFrame` for frontend rendering

**PTY integration (`pty.rs`):**
- Per-pane `SharedScreen` created on spawn, cleaned up on close
- Reader thread feeds raw bytes through `ScreenState::process()` on every read
- Emits `frame-ready-{id}` signal (suppressed during sync output)
- `resize_pty` also resizes the Rust screen buffer
- New `get_frame` IPC command returns latest `ScreenFrame`

**Frontend frame renderer:**
- `CanvasRenderer.drawFrame()` renders a `ScreenFrame` using the existing glyph atlas
- `TerminalHost.drawFrame()` delegates to renderer
- Terminal.tsx listens for `frame-ready` events, fetches frame on RAF via `get_frame`
- JS pipeline stays active for input, selection, search, scrollback
- Both pipelines run in parallel during transition

### 11. Rust Render Mode — JS Pipeline Suppression
Solved the dual-pipeline conflict where the JS text draw and overlay layers were overwriting Rust frames with stale data. Added `rustRenderMode` flag set immediately on terminal mount. In this mode:
- `scheduleTextDraw()` is fully suppressed — no JS canvas draws
- `write()` feeds data directly to the JS parser (for scroll/selection/search state) without scheduling draws
- Overlay URL underline scan skipped (was the source of "lined paper" artifacts)
- Overlay cursor/selection layers still active

### 12. Rust-Native Scrollback
Moved scroll handling entirely to Rust. `ScreenBuffer` now maintains `viewport_offset` and composes visible frames from scrollback history + active screen. Added `scroll_viewport` and `reset_viewport` IPC commands. Wheel events route through `TerminalHost.onScroll` callback to Rust, with immediate frame fetch after scroll. Viewport resets to bottom on user keypress.

### 13. SGR Rendering Fixes
- **Sub-parameter handling**: vte delivers colon-separated sub-params (e.g. `CSI 4:3 m` for curly underline) as grouped arrays. Fixed by iterating `raw_groups` instead of flattened params, preventing `CSI 4:1 m` from being misinterpreted as SGR 4 + SGR 1.
- **Semicolon + colon color forms**: Extended colors now handle both `CSI 38;5;N m` (semicolon, look-ahead across groups) and `CSI 38:5:N m` (colon, single group).
- **Bold-bright promotion**: Palette colors 0-7 with bold now render as bright variant (8-15), matching xterm/Alacritty behavior.
- **Missing SGR codes**: Added blink (5/6/25), double underline (21), overline (53/55).
- **DCH underflow**: Fixed potential crash when delete-characters count exceeds remaining columns.
- **Underline stripping**: Underlines zeroed in `take_frame()` to prevent persistent rendering artifacts.

### 15. Smooth Scrollback Scrolling
Replaced the hardcoded 3-row-jump wheel handler with pixel-accurate scroll accumulation. Trackpad gestures now scroll row-by-row as deltas accumulate past `cellHeight`, giving much finer-grained control when navigating long output from Claude, Gemini, Codex, etc.

- Wheel handler uses raw `e.deltaY` with `deltaMode` handling (pixel, line, page modes)
- `scrollPixelAccum` accumulates sub-row deltas until a full row threshold is crossed
- PageUp/Down and prompt navigation use `setPixelScrollToRow()` for snapped jumps
- Removed `docs/` folder (experimental HTML-in-Canvas design doc, not implemented)

### 16. Scrollback Row Mapping Fix (Root Cause of Overlap Bug)
Fixed a pre-existing bug in the viewport row composition formula that caused scrollback content to overlap active screen content when scrolled. The formula `row - (vpOff - min(vpOff, sbLen))` simplified to `row - 0 = row` (since vpOff is always ≤ sbLen), so viewport rows after the scrollback section mapped to the WRONG active screen rows (e.g., viewport row 10 → active row 10 instead of active row 0).

Fixed to `row - vpOff` in three places:
- `TerminalCore.getRenderState()` → `getRowSource()` (JS rendering)
- `Screen.getCell()` (JS cell access)
- `screen.rs::take_frame()` (Rust frame composition)

This was the actual root cause of the "upper section overlaps lower section" display corruption during scroll.

### 17. Scroll-to-Live Resume Fix
Fixed an issue where scrolling back down to the bottom of output didn't restore the live terminal view — the display stayed frozen on the last JS-rendered scrollback frame until the user clicked outside the terminal. Root cause: when `viewportOffset` reached 0 via wheel scroll, `scheduleTextDraw` bailed early in Rust render mode and no fresh Rust frame was requested. Fix: call `onScroll(0)` (which triggers `get_frame` IPC) whenever scroll returns to `viewportOffset === 0` in Rust render mode. Applied to both wheel scroll and `setPixelScrollToRow` (PageDown/prompt nav).
