## Session 4 — VirtualCanvas Audit Roadmap Execution

### Cross-Reference & Roadmap
- Cross-referenced `daily-growth/Research/virtual-canvas-audit.md` (18 problems, 15 ideas) against current codebase
- Identified what was already resolved vs. remaining work
- Created `daily-growth/roadmaps/virtual-canvas-roadmap.md` — 4-wave implementation plan from current state to audit completion

### Wave 1 — Cleanup
- Removed all Solid.js signal machinery from `VirtualCanvas.ts`: `createSignal` import, `rowVersions`, `rowVersionCounters`, `beginBatch()`, `endBatch()`, `getRowVersion()`, and signal rebuilding in `resize()`
- Removed `beginBatch()`/`endBatch()` calls from `TerminalCore.write()`
- Eliminated per-frame overhead from unused reactive system

### Wave 2 — Scrollback + Graphemes
- **Ring buffer scrollback** (P15, P16, I7): Replaced `ScrollbackRow[]` (25,000 individual typed array allocations at capacity) with a pre-allocated ring buffer — 5 flat typed arrays sized to `scrollbackLimit × cols`. Push is O(1) with zero allocation. Eviction is O(1) via head pointer advance. No more `Array.shift()`.
- **Per-row grapheme storage** (P6, I8): Replaced global `Map<number, string>` (flat index keys) with `rowGraphemes: (Map<number, string> | null)[]` — one optional map per row, keyed by column. `clearRow()` is O(1) for graphemes. Scroll moves map references between row slots — no per-cell iteration. Resize prunes out-of-bounds columns cleanly.
- Updated all consumers: `IRenderer.ts` (`graphemeMap` → `getGrapheme` function), `Renderer.ts`, `TerminalCore.ts`, `TerminalHost.ts` (debug dump), tests

### Wave 3 — Structural Upgrades
- **Row indirection table** (I6): `Uint16Array` rowMap mapping logical→physical rows. Scroll rotates indices instead of copying cell data — zero bytes moved during scroll. `mainRowMap`/`altRowMap` swap with alt screen.
- **Sub-row dirty tracking** (I9): `dirtyColStart`/`dirtyColEnd` per row alongside bitmap. `setCell()` marks only the affected column. Renderer limits all 3 passes (backgrounds, glyphs, decorations) to the dirty column range — typing a single character no longer redraws the entire row.
- **Oversized buffer allocation** (I10): Pages pre-allocated at `max(cols, 320) × max(rows, 100)`. Resize within those bounds requires no reallocation — just update logical dimensions and mark dirty. Eliminates allocation bursts during window drag-resize.
- Updated `Renderer.ts`: all three draw passes take `colStart`/`colEnd` instead of full `cols`, partial row clear targets only the dirty subrange
- Updated `RenderState` interface with `rowOffset`, `dirtyColStart`, `dirtyColEnd`

### Wave 4 — Partial
- **Compact scrollback attributes** (4.2): `sbAttrs` → `Uint16Array` (only 12 bits used), `sbUlColor` dropped entirely (always `DEFAULT_COLOR` in scrollback). Saves ~9.6MB at capacity.
- Items 4.1 (interleaved ArrayBuffer), 4.3 (WASM parser), 4.4 (compressed scrollback + virtual scroll) deferred — depend on WebGL/WASM design decisions
